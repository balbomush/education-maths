# Education Maths (настольное приложение для задач по математике)

Простое офлайн-приложение для Windows, в котором ученик 5–9 классов решает задачи по математике, вводит ответы и сразу получает проверку и обратную связь.

Приложение написано на **Python 3** с использованием стандартной библиотеки **Tkinter** (без внешних GUI-зависимостей).

## Установка и запуск

1. Установить Python 3 (желательно 3.10+), при установке включить пункт «Add Python to PATH».
2. (Опционально) создать виртуальное окружение:

```bash
python -m venv .venv
.venv\Scripts\activate
```

3. Установить зависимости (сейчас проект не требует внешних библиотек, шаг можно пропустить или оставить на будущее):

```bash
pip install -r requirements.txt
```

4. Запуск приложения:

```bash
cd education-maths
python -m app.main
```

Либо:

```bash
python app/main.py
```

## Структура проекта

- `app/` — исходный код приложения:
  - `main.py` — точка входа, инициализация Tk-интерфейса.
  - `core/tasks_loader.py` — загрузка задач из JSON.
  - `core/checker.py` — логика проверки ответов.
  - `core/progress.py` — хранение и обновление прогресса.
  - `ui/` — компоненты пользовательского интерфейса (главное окно, экран задачи, прогресс).
- `data/` — данные:
  - `tasks_5.json`, `tasks_6.json` — задачи для 5 и 6 классов.
  - `progress.json` — файл с прогрессом (создаётся автоматически при первой работе).
- `tests/` — модульные тесты базовой логики (например, проверка числовых ответов).

## JSON-схема задач

Каждый файл `data/tasks_<grade>.json` содержит массив задач. Одна задача имеет следующую структуру:

```json
{
  "id": "5_1_1",
  "grade": 5,
  "topic": "арифметика",
  "text": "Вычисли: 12 + 7",
  "answer_type": "numeric",
  "correct_answer": 19,
  "difficulty": 1,
  "solution_hint": "Сложи десятки и единицы по отдельности",
  "options": null
}
```

Полезные поля:

- `id` — строковый уникальный идентификатор задачи.
- `grade` — класс (целое число от 5 до 9).
- `topic` — тема задачи (например, `арифметика`, `дроби`, `уравнения`, `геометрия`).
- `text` — текст условия задачи.
- `answer_type` — тип ответа:
  - `numeric` — числовой ответ (целые, десятичные, простые дроби вида `1/2`, `3 1/4`).
  - `text` — короткий текстовый ответ.
  - `multiple_choice` — выбор одного или нескольких вариантов.
- `correct_answer` — правильный ответ:
  - для `numeric` — число (или строка, которую можно привести к числу/дроби);
  - для `text` — строка;
  - для `multiple_choice` — массив индексов правильных вариантов, например `[1, 3]`.
- `difficulty` — уровень сложности (1 — легко, 2 — средне, 3 — сложно).
- `solution_hint` — опциональная подсказка или краткий разбор.
- `options` — массив строк с вариантами ответа, используется только при `answer_type = "multiple_choice"`.

### Примеры задач

**Простая арифметика (5 класс):**

```json
{
  "id": "5_1_1",
  "grade": 5,
  "topic": "арифметика",
  "text": "Вычисли: 12 + 7",
  "answer_type": "numeric",
  "correct_answer": 19,
  "difficulty": 1,
  "solution_hint": "Сложи десятки и единицы по отдельности",
  "options": null
}
```

**Дроби (6 класс):**

```json
{
  "id": "6_2_1",
  "grade": 6,
  "topic": "дроби",
  "text": "Сократи дробь 4/8 до несократимого вида.",
  "answer_type": "numeric",
  "correct_answer": "1/2",
  "difficulty": 2,
  "solution_hint": "Раздели числитель и знаменатель на их НОД.",
  "options": null

```

**Множественный выбор:**

```json
{
  "id": "5_3_1",
  "grade": 5,
  "topic": "арифметика",
  "text": "Какое из чисел является чётным?",
  "answer_type": "multiple_choice",
  "correct_answer": [1],
  "difficulty": 1,
  "solution_hint": "Чётные числа делятся на 2 без остатка.",
  "options": ["15", "28", "37", "51"]
}
```

## Логика проверки ответов (коротко)

Логика проверки будет вынесена в модуль `core/checker.py` и будет поддерживать:

- числовые ответы с учётом допустимой погрешности для вещественных чисел;
- разбор простых дробей и смешанных чисел (`3 1/4`);
- проверку вариантов множественного выбора (`multiple_choice`);
- нормализацию и сравнение коротких текстовых ответов.

## Прогресс и подбор задач

- Прогресс хранится в файле `data/progress.json` и содержит результаты по каждой задаче и тему.
- Подбор задач опирается на выбранный класс, тему и (в перспективе) успешность в этой теме.
